# Funding Rate News Bot

Телеграм-бот, который следит за официальными объявлениями бирж (Binance, Bybit,
OKX, KuCoin, Bitget, Gate.io) и присылает в чат/канал уведомления, когда меняется
формула/параметры funding rate — глобально (для всей биржи) или локально
(для конкретной монеты/пары).

## Как это работает

1. Каждые `POLL_INTERVAL_SECONDS` секунд бот опрашивает "коллекторы" —
   по одному модулю на биржу (`collectors/*.py`).
2. Все свежие объявления фильтруются по ключевым словам, связанным с funding
   rate (см. `FUNDING_KEYWORDS` в `config.py`).
3. Прошедшие фильтр новости классифицируются как **глобальные** (затрагивают
   всю биржу / все пары) или **локальные** (перечислены конкретные тикеры) —
   логика в `classifier.py`.
4. Уже отправленные новости запоминаются в SQLite (`db.py`), чтобы не дублировать.
5. Итоговое сообщение уходит в чат/канал, указанный в `CHAT_ID`.

## Установка

```bash
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Откройте `.env` и заполните:

- `BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather).
- `CHAT_ID` — ID чата/канала, куда слать уведомления.
  - Для публичного канала можно использовать `@channelusername` — но проще
    получить числовой ID (обычно начинается с `-100...`). Добавьте бота в
    канал/чат **администратором**, отправьте туда любое сообщение и
    воспользуйтесь любым ботом-хелпером (например `@getidsbot`) чтобы узнать ID.
- `EXCHANGES` — через запятую какие биржи включены (по умолчанию все шесть).
- `POLL_INTERVAL_SECONDS` — как часто опрашивать источники (не делайте
  слишком маленьким, чтобы не словить рейт-лимиты).

## Запуск

```bash
python bot.py
```

Бот сразу сделает первый опрос всех источников, затем продолжит опрашивать
их по расписанию.

## Важные технические детали и ограничения

- **Bybit, OKX, KuCoin** — используются официальные публичные REST API
  объявлений, авторизация не требуется. Это самые надёжные источники.
- **Binance** — официального REST API для объявлений нет. Используется тот
  же служебный (недокументированный) endpoint, которым пользуется сам сайт
  Binance. Он может измениться без предупреждения — если Binance перестанет
  присылать новости, поправьте `ANNOUNCEMENT_URL`/`CATALOG_IDS` в
  `collectors/binance.py`.
- **Bitget и Gate.io** — у этих бирж вообще нет публичного API для
  объявлений, поэтому используется скрейпинг HTML-страниц (поиск/лента
  объявлений). На практике подтвердилось, что список объявлений на обеих
  страницах подгружается через JavaScript уже в браузере пользователя — в
  "сыром" HTML, который получает бот, ссылок на статьи просто нет (при 0
  найденных новостей в логе будет видно предупреждение с размером
  полученного HTML). Обойти это можно рендерингом страницы через headless-
  браузер (Playwright/Selenium) вместо простого HTTP-запроса — это более
  тяжёлое решение, которое можно добавить отдельно при необходимости. Пока
  что для этих двух бирж рекомендуется либо смириться с тем, что уведомлений
  оттуда не будет, либо временно убрать их из `EXCHANGES` в `.env`.
- Ключевые слова для фильтрации (`FUNDING_KEYWORDS` в `config.py`) можно
  и нужно донастраивать под себя — например, добавить конкретные тикеры,
  которые вас интересуют, или убрать шум.
- Классификация "глобально vs локально" (`classifier.py`) сейчас основана на
  простых эвристиках: наличие маркеров вида "all pairs"/"all contracts" →
  глобально; наличие конкретных тикеров в тексте → локально. При желании
  можно улучшить (например, считать глобальным только явные маркеры, а не
  "нет тикеров = глобально").

## Расширение

Чтобы добавить ещё одну биржу:
1. Создайте `collectors/<exchange>.py` с классом-наследником `BaseCollector`,
   реализующим `async def fetch(self, session) -> List[NewsItem]`.
2. Зарегистрируйте класс в `COLLECTOR_REGISTRY` в `collectors/__init__.py`.
3. Добавьте имя биржи в `EXCHANGES` в `.env`.

## Деплой на сервер (кратко)

Проще всего запустить как systemd-сервис или в Docker/screen/tmux, чтобы
процесс работал постоянно. Пример systemd unit-файла:

```ini
[Unit]
Description=Funding Rate Bot
After=network.target

[Service]
WorkingDirectory=/path/to/funding_rate_bot
ExecStart=/path/to/funding_rate_bot/venv/bin/python bot.py
Restart=always
RestartSec=5
EnvironmentFile=/path/to/funding_rate_bot/.env

[Install]
WantedBy=multi-user.target
```
